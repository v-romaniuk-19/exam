# Event-Driven Architecture: EMQX MQTT Broker

Цей проєкт демонструє Event-Driven архітектуру за допомогою розгортання брокера повідомлень EMQX та двох веб-клієнтів, що взаємодіють у реальному часі.

## Як запустити

1. **Запустіть MQTT брокер EMQX:**
   В корені проєкту виконайте команду:
   \`\`\`bash
   docker-compose up -d
   \`\`\`
   *Брокер буде доступний на портах 1883 (TCP) та 8083 (WebSockets). Панель керування: http://localhost:18083 (admin / public).*

2. **Запустіть веб-клієнти:**
   Встановіть залежності та запустіть локальний сервер:
   \`\`\`bash
   npm install
   npm start
   \`\`\`
    * Web 1: http://localhost:8080/web1/
    * Web 2: http://localhost:8080/web2/

## Демонстрація крос-протокольної взаємодії (Postman)

Браузери використовують протокол **MQTT over WebSockets** (порт 8083). Щоб продемонструвати крос-протокольність, ми використаємо Postman через стандартний **MQTT over TCP** (порт 1883).

1. Відкрийте Postman (версія v10+ має підтримку MQTT).
2. Створіть новий запит **MQTT**.
3. URL підключення: `mqtt://localhost:1883`. Натисніть **Connect**.
4. У вкладці **Topics** підпишіться на топік: `exam/messages`.
5. У вкладці **Message** введіть топік `exam/messages` та відправте JSON:
   \`\`\`json
   { "sender": "Postman", "text": "Привіт з протоколу TCP!" }
   \`\`\`
6. Перейдіть до браузерів: ви побачите, що повідомлення від Postman миттєво з'явилося у `Web 1` та `Web 2`. Відповіді з браузерів також з'являться у Postman.
